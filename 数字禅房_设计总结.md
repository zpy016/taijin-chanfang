## 二、设计原则（通用 7 条 + v9 新增 1 条 = 8 条）

> 在 v7 的 5 条原则基础上，v8 沉淀了 2 条新增原则，v9 再补 1 条。

1. **阅读为本，工具克制**：阅读区是主舞台，工具栏、目录、底栏都要让位给文字。
2. **新中式禅意，但不复古拘谨**：
   - 主色：`--accent:#8B6846`（棕）；
   - 纸面：`--bg:#F8F3EA` / `--paper:#FFFDF7`；
   - 衬线体：`Songti SC / STSong / Noto Serif CJK SC`，承担标题、引用、装饰大字；
   - 无衬线：`-apple-system / PingFang SC / Microsoft YaHei`，承担正文段落与 UI 控件；
   - **全文最多两种字体族**，通过字号 / 字重 / 颜色构建层级，而非靠字体跳变。
3. **太晋先生引用是"仪式空间"**：所有 `<blockquote>` 统一为 `.taijin-quote` 规范——衬线、居中（桌面）、柔暖底色、署名独立行 `——太晋先生`，不混入正文样式。
4. **昼夜双模可读性**：夜间模式色板独立配色，对所有涉色组件（如循环图、抽屉、pill tab、搜索框）显式做 `body.night` 覆盖，不靠继承。
5. **结构外显**：四个维度配色不同，正文区块顶部有 `section-kicker` 序号 + 维度名，构建"章—节—段"可感知层级；DOM 通过 `data-dim / data-tone / data-color` 自描述。
6. **【v8 新增】同语义层级 → 同组件**：当两段文字在叙事上是同等重要的"小结类"，其视觉权重必须一致。例：内循环底部的"回路"与外循环底部的"反哺关系"，统一为 `.cycle-feedback`。
7. **【v8 新增】像素级品味，不放过细节**：进度线下移 6px、搜索框双 ×、4 个圆角不一致、字号按钮大小 A 不齐 —— 这些"小问题"才是品质的分界线。
8. **【v9 新增】CSS 隐式行为必须显式审计**：`overflow-x:hidden` 会隐式创建滚动容器（使 `position:sticky` 失效）；`touch-action` 不隔离会导致手势冒泡；`viewport` 不锁缩放会被双指捏爆。这些不是"边角案例"，而是"一行 CSS 决定整个交互是否成立"的系统性风险。**每次涉及 overflow / touch-action / viewport 的修改，必须同时检查其对 sticky、滚动容器链、手势隔离的影响。**

### 4.3 左侧阅读路径（v7.6+ sticky，v9 根因修复）

> **对应 `数字禅房v9.html` 第 40 行 `.left-rail` sticky 声明 + 第 10 行 `overflow-x:clip` 修复。**

**v9 根因修复（核心经验）**：

v7.6 已加 `position:sticky`，但桌面端左栏仍然跟随滚动。根因不在 `.left-rail` 本身，而在 `html` 和 `body` 均设了 `overflow-x:hidden`：

```css
/* v8 及之前 — 导致 sticky 失效 */
html{overflow-x:hidden}
body{overflow-x:hidden;max-width:100vw}

/* v9 修复 — clip 不创建滚动容器，sticky 正常工作 */
html{scroll-behavior:smooth;overflow-x:clip}
body{overflow-x:clip;max-width:100vw}
```

**CSS 规范知识点**：当元素在任一轴上设置 `overflow:hidden/auto/scroll`，该元素会成为**滚动容器**。`overflow-x:hidden` 只设了一个轴，CSS 规范会将另一个轴隐式设为 `auto`，进一步确保 html/body 被识别为滚动容器。其后代 `position:sticky` 元素的粘附参照物从**视口**变成了**该容器**——由于 html/body 高度远大于视口，sticky 视觉上完全失效。

`overflow:clip` 同样阻止横向滚动条出现，但**不会创建滚动容器**，是"防横滚 + 保 sticky"的标准方案。

sticky 元素本身的完整声明（v9 第 40 行）：

```css
.left-rail{
  position:sticky;top:84px;
  border-radius:18px;
  overflow-x:hidden;overflow-y:auto;
  overscroll-behavior:contain;
  -webkit-overflow-scrolling:touch;
  scrollbar-width:thin;
  scrollbar-color:color-mix(in srgb,var(--accent) 28%,transparent) transparent;
  max-height:calc(100vh - 104px);
  align-self:flex-start  /* 防止 grid 的 stretch 撑满整列导致 sticky 失效 */
}
```

其他左栏特性（沿用 v7.4–v7.6）：
- 暖色卡板背景：`linear-gradient(...)`，与正文纸面相近但稍"亮"。
- 左缘 3px 主色竖线，强化卡板形态。
- 四角统一 `border-radius:18px 18px 18px 18px`。
- 独立滚动：`overflow-y:auto;overscroll-behavior:contain`。
- 目录项双层结构：`.toc-link-title`（标题深色 13px 加粗）+ `.toc-link-sub`（副标题浅色 12px），均不折行。
- 当前章节高亮：左→右淡棕渐变 + 主色文字。
- IntersectionObserver 同步 `.toc-links a.active` 与底栏 `.tab-link.active`。

### 5.2 v9 移动端三大修复（对应 `数字禅房v9.html`）

> v8 在 iPhone 微信内测时发现 3 个问题，v9 全部修复。

#### 修复 1：滑动冲突 + 横向滚动条

**症状**：在 iPhone 微信中向下滑动时，底部选项卡被触发横滑；页面存在横向滚动条。

**根因**：
- `html` 和 `body` 的 `overflow-x:hidden` 未能完全阻止微信内置浏览器的横滚（v9 改为 `overflow-x:clip`）；
- 移动端 `body` 未限制 `touch-action`，垂直滑动事件可能冒泡到横滑容器；
- `.mobile-tab` 用 `position:sticky` 或非 fixed 定位，与页面滚动竞争；
- `.perspective-grid` 的横滑 `touch-action` 未隔离，垂直滑动可能触发卡片横滑。

**修复**（v9 第 5、10、119、161 行）：
```css
/* viewport 禁缩放 */
<meta name="viewport" content="width=device-width, initial-scale=1.0, maximum-scale=1.0, user-scalable=no, viewport-fit=cover">

/* html/body 防横滚不创建滚动容器 */
html{overflow-x:clip}
body{overflow-x:clip}

/* 移动端 body 限制垂直滑动 */
@media (max-width:820px){
  body{touch-action:pan-y}
}

/* 底栏固定底部 */
.mobile-tab{position:fixed;bottom:0;left:0;right:0;z-index:40}

/* 横滑容器隔离手势 */
.perspective-grid{touch-action:pan-x}
```

#### 修复 2：双指缩放未被禁止

**症状**：用户可以双指捏合缩放页面，破坏阅读体验。

**根因**：viewport meta 缺少 `maximum-scale=1.0, user-scalable=no`。

**修复**（v9 第 5 行）：
```html
<meta name="viewport" content="width=device-width, initial-scale=1.0, maximum-scale=1.0, user-scalable=no, viewport-fit=cover">
```

#### 修复 3：字体调大反而感觉变小

**症状**：用户调大字号后，可见区域变窄，字体感觉反而变小了。

**根因 A（布局）**：移动端 `.left-rail` 未隐藏，仍占据 grid 一列宽度，挤压阅读区。
**修复**（v9 第 277 行）：
```css
@media (max-width:820px){
  .left-rail{display:none!important}
  .shell{grid-template-columns:1fr!important}
}
```

**根因 B（排版）**：大量固定 px 字号不响应 `--font-step` 变量，调大字号时这些元素不联动。
**修复**：v9 将所有固定 px 字号统一改为 `calc(Xpx + var(--font-step))`，覆盖元素包括：
- `.reader-section` 正文（`calc(16px + var(--font-step))`）
- h2 / h3（`calc(26px + var(--font-step))` / `calc(19px + var(--font-step))`）
- `.taijin-quote-line`（`calc(17px + var(--font-step))`）
- `.research-meta` label / value（`calc(17px + var(--font-step))`）
- `.cycle-node strong`（`calc(16px + var(--font-step))`）
- `.cycle-feedback`（`calc(13px + var(--font-step))`）
- `.dimension-card span / strong / em`（全部联动）
- `.concept-title-main / -sub`（全部联动）

## 八、关键技术决策与心法

### 8.1 进度条不能"飘"，必须贴 topbar 底边

**最终答案**：进度条做成 topbar 的子元素 `position:absolute;bottom:-1px;height:2px`，**直接替代 topbar 的下边线**。桌面、移动同位。

### 8.2 搜索框：所见即所得，不要按钮 → 浮层 二段式

- 用户看到搜索框 = 立刻能输入，不需要"先点按钮再输入"；
- 浏览器原生 cancel × 一定要隐藏（多浏览器多伪元素，需要一组）。

### 8.3 字号控件：单按钮 Aa + 浮层 +/−，不要并排两个 A

不同字号的两个 A 视觉永远不可能整齐。最终用单按钮 `Aa` + pill 浮层 `[ −  当前档位文字  + ]`。

### 8.4 章节抽屉的两个铁律

1. **不能遮挡底栏**：`max-height:calc(100vh - 64px - safe-area - 12px)` + `margin-bottom:64px`；
2. **关闭后必须能滚动到目标章节**：capture 阶段 `preventDefault + stopPropagation`，先 `closeTabSheet()` 再 setTimeout 220ms 后 `window.scrollTo`。

### 8.5 移动端 hero 居中：4 个细节

1. `.hero` 是 `display:grid` 容器，单 `text-align:center` 不够，必须叠 `justify-items:center`；
2. 子元素需要 `display:block; margin:0 auto`；
3. 中文 `letter-spacing` 末尾留白，需要等量 `text-indent` 补偿；
4. hero 副本（`<p>`）必须 `display:none`。

### 8.6 同语义 → 同组件

内循环底"回路"与外循环底"反哺关系"是同等地位的"循环小结"，必须同组件 `.cycle-feedback`。

### 8.7 公式语言：数学等式过冷，思想类内容改白话

禅意 / 治愈 / 心法类内容里，等式公式让人疏离，白话表达让人共鸣。

### 8.8 移动端"太靠左 / 拥挤"5 步诊断流程

1. 容器 padding 是否对称；
2. flex/grid item 是否真的居中；
3. `text-align` 是否在 grid item 上；
4. letter-spacing 偏移补偿；
5. 字数 vs 字号。

### 8.9【v9 新增】overflow-x:hidden 是 sticky 的隐形杀手

**完整诊断链**：

```
症状：桌面端左侧目录栏跟随页面滚动，sticky 不生效
  ↓
初步排查：.left-rail 确实有 position:sticky; top:84px; max-height:calc(100vh-104px); align-self:flex-start
  ↓
深入排查：检查 html / body 是否设了 overflow-x:hidden
  ↓
根因确认：overflow-x:hidden 使 html/body 成为滚动容器，sticky 的参照物从视口变为该容器
  ↓
修复：overflow-x:hidden → overflow-x:clip
  ↓
验证：左栏固定不动，sticky 生效
```

**通用心法**：遇到 `position:sticky` 不生效，按以下顺序排查：
1. 元素自身是否有 `position:sticky` + `top` + `max-height` + `align-self:flex-start`（grid 场景）；
2. 祖先元素是否有 `overflow:hidden/auto/scroll`（任一轴）—— 这是最常见的隐形杀手；
3. 祖先元素是否有 `transform/filter/will-change`（会创建新的包含块）；
4. 元素是否在 grid/flex 容器中被 `stretch` 拉满（需要 `align-self:flex-start`）。

### 8.10【v9 新增】touch-action 手势隔离

移动端存在多种手势竞争：页面垂直滚动、横滑卡片水平滑动、底栏 tab 点击。不隔离会导致：
- 垂直滑动触发卡片横滑；
- 横滑卡片时页面也跟着滚。

**解法**：
- `body{touch-action:pan-y}` — 限制 body 只响应垂直滚动；
- `.perspective-grid{touch-action:pan-x}` — 横滑容器只响应水平滑动，阻止垂直手势冒泡。

## 十一、内容沉淀与品味偏好

### 11.1 视觉品味

1. **不要花哨**：摒弃过度渐变、华丽阴影、emoji 装饰、纯色实底大块背景。喜欢 1px 主色细线、克制的浅色软底、衬线节奏。

3. **字数 vs 字号**：≤9 字的标题/品牌名必须视觉居中；长段落保持左对齐。
4. **重心居中**：移动端首屏 hero 必须重心居中；2×2 卡片必须左右等距。
5. **单元图标要语义化**：禅意 / 山-日 / 罗盘 / 眼-真 / 勾-上行，要与章节内容呼应。

### 11.2 交互品味

6. **所见即所得**：搜索框就是搜索框，不要按钮 + 浮层；导航就是直接列表，不要二级菜单。
7. **抽屉 / 浮层必须能关**：点遮罩、点 ×、点列表项、ESC 都能关；锁滚动后必须能滚到目标。
8. **首屏可达每章**：移动端 2×2 章节卡，第一屏直接跳到任一章。
9. **同语义 → 同组件**：用户对"同等关系但视觉权重不同"零容忍。

### 11.3 内容品味

10. **思想类内容改白话**：禅意 / 治愈类，等式公式 → 白话表达。
11. **写错的内容比设计 bug 优先级更高**。
12. **太晋先生引用是仪式**：必须独立组件 `.taijin-quote`。

### 11.4 协作品味

13. **要可决策**：涉及 UI/UX/技术架构的决定，必须给 2–3 个可比方案 + 推荐项，编号 A1/A2/B1...。
14. **报告结构感强**：先结论，再分项，再注意事项。回答开头不要"好的"。
15. **遇到不确定，直接说**。
16. **小步迭代**：一次最多落地 4–8 个改动，每次给出小版本号；版本号沉淀变更历史。

## 十三、未来更新建议

如果将来要继续优化，按以下顺序考虑：

1. **完整字号档位测试**：极小档（-2）下 dimension-card / cycle-node 是否可读；最大档（+5）下抽屉 / pill tab 是否溢出。
2. **键盘导航无障碍**：抽屉 + Aa 浮层 + 搜索框的 Tab 焦点链路是否合理；ESC 是否覆盖所有浮层。
3. **打印样式 `@media print`**：长文打印时去掉左栏、底栏、抽屉、搜索框、进度线，正文继承 serif。
4. **章节书签 / 阅读位置记忆**：`localStorage` 记录最后阅读节，下次打开直接定位。
5. **PWA 化（可选）**：加一个 manifest + service worker，让 iOS Safari "添加到主屏幕"后体验更接近原生。
6. **图谱模块的横滑动手势**：六节点循环图在小屏上是否可双指缩放、拖动查看。

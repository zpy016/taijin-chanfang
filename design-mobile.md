# 数字禅房 · 手机版（微信内嵌浏览器）设计文档

## 目标
在 iPhone（微信内嵌浏览器）中打开时，提供美观、方便、不打折扣的阅读体验。

## 屏幕假设
- 视口宽度：375px ~ 430px（iPhone 标准 / Plus / Pro Max）
- 高度：~700px ~ 900px
- 输入方式：手指触摸，无 hover
- 浏览器：微信 WKWebView，支持现代 CSS

## 设计原则
1. **内容优先**：手机屏小，一切非内容元素必须最小化或隐藏
2. **单栏阅读**：正文全宽，无左右边距浪费
3. **底部操作**：重要操作放在底部（拇指可及区域）
4. **抽屉替代覆盖**：右侧面板在手机上改为底部弹出，不覆盖正文
5. **滚动优化**：路线图改为水平可滚动，避免一行6个卡片被压扁

## 各区域手机版改造方案

### 1. 顶部导航栏
- **现状**：6个章节链接 + 品牌名 + 夜间模式按钮，在手机上溢出
- **手机版**：
  - 保留品牌名"如是·觉醒·改命"（左侧，缩小字号）
  - 移除中间的6个章节链接（改为底部Tab栏）
  - 保留夜间模式按钮（右侧）
  - 高度不变（56px），但去除多余的章节导航

### 2. 左侧栏（章节图标栏）
- **现状**：固定在左侧，52px宽，垂直排列6个图标+标签
- **手机版**：
  - **完全隐藏**，改为**底部固定Tab栏**
  - 底部Tab栏：52px高，固定底部，白色背景，顶部1px分隔线
  - 6个Tab：图标+文字，每个约60px宽，均分底部
  - 点击Tab → 打开底部章节面板（内容导航）
  - 当前激活的Tab高亮显示

### 3. 左侧面板（内容导航面板）
- **现状**：从左侧滑出，360px宽
- **手机版**：
  - 改为**底部抽屉式面板**（类似iOS Action Sheet）
  - 从底部向上滑出，占屏幕高度约70%
  - 顶部有拖拽指示条（一个小灰线）
  - 面板内可滚动
  - 点击遮罩或下滑 → 关闭
  - 面板标题居中，字号稍大

### 4. 右侧面板（笔记、搜索等）
- **现状**：从右侧滑出，或全屏模态框
- **手机版**：
  - 搜索面板：全屏覆盖，从底部向上滑出
  - 笔记面板：全屏覆盖，或底部70%高度
  - 工具按钮：改为右下角浮动按钮（FAB），点击展开菜单

### 5. 右侧工具栏（浮动按钮组）
- **现状**：右侧居中浮动，6个圆形按钮
- **手机版**：
  - 改为**右下角浮动按钮**（FAB），48px圆形，带+号图标
  - 点击后展开5个选项（注释、搜索、笔记、历史、工具）
  - 展开时半透黑色背景遮罩
  - 在底部Tab栏上方，不冲突

### 6. 主内容区域
- **现状**：padding: 0 40px，margin-left: 52px，max-width: 1200px
- **手机版**：
  - 移除所有 margin-left
  - padding: 0 16px（左右更紧凑）
  - 宽度100%
  - 底部padding增加到100px（为底部Tab栏留空）
  - 字体：16px（稍大，适合手机阅读）
  - 行高：1.8（保持）
  - 标题字号适当缩小：h1 1.6rem，h2 1.3rem，h3 1.1rem

### 7. 路线图（poster-flow）
- **现状**：6张卡片 + 箭头，flex nowrap，一行显示
- **手机版**：
  - 改为**水平滚动容器**（`overflow-x: auto`）
  - 卡片宽度：140px，固定不可压缩
  - 箭头缩小为点或隐藏
  - 每个卡片间距12px
  - 用户可左右滑动浏览6个卡片
  - 提示：可左右滑动（可选，小字提示）

### 8. 引用块（blockquote）
- 手机版保持，但 padding 缩小到 14px 18px

### 9. 表格（Markdown表格）
- 如果内容过宽，改为横向滚动（`overflow-x: auto`）
- 字体缩小到 14px

### 10. 夜间模式
- 保持现有逻辑，手机版同样支持
- 切换按钮在顶部导航栏右侧

### 11. 阅读进度条
- 保持顶部固定，不变

### 12. 触摸优化
- 所有按钮/链接最小44px高/宽（iOS人机界面指南）
- 增加 tap-highlight-color: transparent
- 增加 active 状态的触摸反馈（颜色变化）
- 左右滑动切换章节（可选增强）

## 技术实现

### 响应式断点
- 桌面版：`@media (min-width: 769px)` → 保持现有所有样式
- 手机版：`@media (max-width: 768px)` → 覆盖为手机样式

### 结构修改（HTML）
1. 底部Tab栏（新增HTML结构，放在 body 末尾，position: fixed bottom: 0）
2. 底部抽屉面板（用现有 leftPanel，但手机版改变样式和定位）
3. 右下角FAB按钮（新增，放在 body 末尾）
4. 移动菜单按钮（已有，但需要调整样式）

### JS 修改
1. 检测手机环境（`window.innerWidth <= 768`），调整行为
2. 底部Tab栏点击 → 打开底部面板
3. 底部面板关闭：点击遮罩、点击关闭按钮、下滑手势
4. FAB 展开/收起逻辑
5. 路线图滚动联动（可选，自动滚动到当前章节对应的卡片）

---

## 视觉风格
- 保持现有暖色调（米白+咖啡棕），不切换新配色
- 保持思源宋体/黑体字体栈
- 手机版更简洁，留白适当减少
- 阴影和模糊效果在微信中适度减少（性能考虑）

---

## 交互流程（手机版）

1. **打开页面**：看到总览内容 + 底部Tab栏（总览高亮）
2. **阅读总览**：正常滚动，底部Tab栏固定
3. **切换到道法溯源**：点击底部Tab栏"溯源" → 正文直接跳转到维度一 + 底部面板打开显示该章节内容导航
4. **查看内容导航**：在底部面板中点击"总起：道法溯源" → 正文跳转到该章节开头
5. **打开工具**：点击右下角FAB → 展开5个选项 → 点击搜索/注释等
6. **切换夜间模式**：点击顶部导航栏右侧月亮图标
7. **关闭面板**：点击遮罩或下滑关闭
